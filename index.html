CSS Flexbox

I have chosen to only implement CSS-Flexbox as it enabled more inter dynamic element widths without being too constrained.

Since I have implemented a row wise element style, using flexbox for each row containing multiple elements in only one dimension made sense

If I wanted to have divs with multiple rows inside I would use grid.

HTML Canvas

Not time based?? Had to experiment with where the objects ended up in a certain time due to difference in travel lengths in the animation.

In comparison with SVG, you have to clear the whole image and continously redraw all the figures/frames

Have to move a "cursor" to draw by drawing lines or arcs. Instead of SVG that has a better API with existing shapes like lines, circles or polygons where you just have to supply coordinates of the corners

By having complete control over every frame that is drawn, the animation can easily be paused by not requesting the next frame. This way the still image of where the animation left off is showed.

If the animations can move independently of each other in terms of time, canvas is a great choice.

SVG

Based on time, could set 10 seconds and the whole animation conformed to that time interval

Do not have to redraw the figures, just change the parameters of the figures

When the animation is stopped, the image is returned to the original image and not where the animation left off. To fix this the drawing should be manually redrawn and the coordinates of where the animation left off should be calculated by elapsed time.

If the animations are time sensitive, this is the best option
